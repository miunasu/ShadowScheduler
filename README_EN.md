# WPTaskScheduler-PoC

[中文版本](README.md)

Proof of Concept for creating **hidden scheduled tasks** via the undocumented Windows WP TaskScheduler RPC interface.

## Background

Windows 10 introduced an internal scheduling subsystem called **WP TaskScheduler** (`WPTaskScheduler.dll`), running inside the Task Scheduler service. It exposes an undocumented RPC interface:

```
Interface: {33D84484-3626-47EE-8C6F-E7E98B113BE1} v2.0
Transport: ncalrpc (Local RPC)
```

Tasks created through this interface are **completely invisible** to:
- `schtasks /query`
- `Get-ScheduledTask`
- Task Scheduler MMC snap-in
- `TaskCache\Tree` / `TaskCache\Tasks` registry
- `C:\Windows\System32\Tasks\` filesystem


Standard investigation results:
| Investigation Method | Result | Reason |
|---------------------|--------|--------|
| `schtasks /query` | Not found | Relies on TaskCache\Tree registry enumeration |
| `Get-ScheduledTask` | Not found | Uses ITaskService COM interface underneath |
| Task Scheduler MMC | Invisible | Same as above |
| `C:\Windows\System32\Tasks\` directory | No XML | Task not written to filesystem |
| TaskCache\Tree registry | No entry | NdrClientCall3 bypasses Tree registration |
| TaskCache\Tasks GUID enumeration | No `{e0bff96f...}` | Task not written to Tasks subkey |
| TaskCache\Plain/Logon/Boot categories | No matching GUID | No category registration |
| Run/RunOnce registry | Clean | Not used |
| Startup folder | Clean | Not used |
| WMI event subscriptions | System defaults only | Not used |
| Windows Services | No malicious service | Not used |
| IFEO debugger hijack | Clean | Not used |
| Winlogon Shell/Userinit | Normal values | Not used |
| COM hijack (HKCU CLSID) | OneDrive only | Not used |
| AppInit_DLLs | Empty/Disabled | Not used |
| LSA/Print/Network Provider | Clean | Not used |
| autoruns | Clean | All mechanisms bypassed, autoruns blinded |

Persistence data is stored only in:
```
HKLM\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Schedule\WP\TaskScheduler\Schedules\
```

## Two Implementations

### raw_ndr64 - Shellcode-Ready

Manually reversed NDR64 procedure/type format strings, calling `WptsCreateSchedule` via `NdrClientCall3`.

- **No external dependencies** beyond ntdll/rpcrt4 (already loaded in any Windows process)
- Self-contained RPC binding and marshaling
- Suitable as a shellcode foundation or position-independent payload
- Reversed directly from malware sample

### midl_poc - MIDL Stub

Standard MIDL-generated client stub from IDL. Cleaner code, easier to modify, but requires the generated stub files.

## Build

Requires MSVC (Visual Studio Build Tools x64):

```batch
:: raw_ndr64 version
cd raw_ndr64
cl /W3 /nologo /O2 poc_wp_task.c /link rpcrt4.lib ole32.lib

:: midl_poc version
cd midl_poc
midl /protocol all /client stub /server none /env x64 /app_config WPTaskScheduler.idl
cl /W3 /nologo /O2 poc_midl.c WPTaskScheduler_c.c /link rpcrt4.lib ole32.lib
```

## Usage

```
raw_ndr64\poc_wp_task.exe <executable_path> [arguments]
midl_poc\poc_midl.exe <executable_path> [arguments]
```

Creates a logon-triggered hidden task named `ShadowScheduler_<GUID>` that executes the specified program on user login.
